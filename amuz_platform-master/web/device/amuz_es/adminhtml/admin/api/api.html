get /ws.php?action=cv&couponcode=playmp4&medianame=basha
Coupon code validation (ws.php?action=cv&couponcode=playmp4&medianame=bashaGet)
user entered coupon code is validated with cheat code and series of coupon codes generated randomly. \nCurrently for that particular media to the user it is opened for 24 hours. user can able to watch any number of times without 2nd coupon<br/>\nEstimated parameters couponcode and title of mediafile\n

Parameters Explanation

couponcode is the coupon number which the user will get physically to watch the premium content

medianame is the title of medifile which will get in the response on clicking the each channel

Responses

response_code: If the response code is 1, user entered a valid coupon or cheatcode, if response_code is 0 user entered a invalid copon

response: response is like a description for the response_code for our understanding, if response is zero response will have the reason why it failed

coupontype: if coupon type is 1, user can able to watch every media content, coupon type 2, user should able to watch only particular media clicked

validatetill: valiidity date expired of that coupon, if it execeeds user should not gain watch that video

call to piwik log analytics (ws.php?action=pi&phonenum=8867895919&useragent=firefox&mediapath=directorypathofmedia&medianame=mediatitle&adname=adname&adpath=directorypathofadGet)
user entered coupon code is validated with cheat code and series of coupon codes generated randomly. \nCurrently for that particular media to the user it is opened for 24 hours. user can able to watch any number of times without 2nd coupon<br/>\nEstimated parameters couponcode and title of mediafile\n

Parameters Explanation

phonenum if the user is registered, send the user registered mobile number if not send empty

useragent these parameter is to capture the user device information like manufacturer and os and tablet or mobile or phone

adpath is the json object adpath in the previous response

mediapath is the json object mediapath in the previous response

adname currently we are not maintaining any advertisement names, send same as the adpath

medianame is the json object title in the previous response

couponcode if user used coupon for playing the particular content send that code, otherwise send empty

Responses

if success, Empty Response

if failed, response_code will be 0 and description will be in response.

get /ws.php?action=rg&phonenum=8867895919
registration of user with phone number (ws.php?action=rg&phonenum=8867895919Get)
if response_code is 1, user is already existed in database and \nif response_code is 2, new user and OTP will come as an response, after entering the otp the action=vp starts\n

Parameters Explanation

phonenum is the user entere mobile number for registration

Responses

If the response code is 1, user will get a otp to his registered mobile number and in object response you will get an otp which have been sent to the registered mobile number and in senderid object u will get a senderid phonenumber to pick the otp automatically, UUID helps to check the delivery report of the sms

If the response code is 0, sending sms to registered mobile number is failed. reason you will get in object response

If the response code is 2, user is registerd with amuz but he dint validated otp in response you will get otp sent to his mobile previously

if response_code is 1, phone number is validated\nif response_code is 0, user entered invalid otp\n device will be 0 if sms send via modem of the device and it will be 1 if sms sent via cloud api

phonenum is the user entere mobile number for registration

Responses

If the response code is 1, user entered otp is successfully validated with the mobile number

If the response code is 0, invalid otp or description will be available in response
